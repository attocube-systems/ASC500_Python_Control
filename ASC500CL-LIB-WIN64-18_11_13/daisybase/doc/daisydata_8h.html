Detailed Description

Service functions for data handling on top of daisybase.h.

Defines functions that simplify the use of the daisybase library - primarily the data handling. They allow to avoid callback functions (DYB_setDataCallback, DYB_setEventCallback) (and thus the need of thread synchronisation) in application programs.

These functions must not be called from the context of a daisybase callback function.

Int32  size

Configure Data Channel Buffering.

The function configures if data arriving from a specific data channel are buffered and sets the default size of the buffer.

If the default size is set to 0, data are not buffered and data callback functions of daisybase (DYB_setDataCallback) can be used.

If it is set to a positive value, the data are buffered and can be retreived with DYB_getDataBuffer. The actual value of the size is relevant only for data channels that are triggered by timer; in all other cases the "native" buffer size is used. If size is too small (< 128), timer triggered data will not be buffered to avoid too mucht buffer-full events.

If buffering is enabled, no data callback function can be used for the channel.

Parameters
channelNumber of the channel of interest (0..13)
sizeBuffer size [32 bit items]
Returns
DYB_Ok - Success
DYB_OutOfRange - size too big (> 1M) or negative

Retrieve Data Channel Buffer.

If a data channel is configured for buffering with DYB_configureDataBuffering, the next buffer can be retrieved with this function without using data callback functions.

Normally, only completely filled buffers are returned and an error DYB_OutOfRange is signalled when no full buffer is available. No data will be returned twice. The user can change this behaviour by requesting also partially filled buffers with the parameter fullOnly = 0. The partially filled buffer may be returned multiple times until it is full. In the case of scanner triggered data, a frame is considered full when the upmost OR the lowermost line has been scanned.

A data frame is available when it is complete until it is retrieved or the next frame is complete. If it is not retrieved in time, the frame number may "jump".

Parameters
channelNumber of the channel of interest (0..13)
fullOnlyIf only completely filled buffers are requested.
frameNoOutput: Number of the frame. With fullOnly=0 the same frame can be returned repeatedly.
indexOutput: Index of the first element in the buffer.
dataSizeInput: Size of the data buffer provided by the user. If insufficient, DYB_OutOfRange will be returned. Output: Number of valid data (32-bit items) in the buffer.
dataOutput: Pointer to an array to store the data. The array must be provided by the caller and its size must be at least one frame size (DYB_getFrameSize).
metaOutput: Pointer to a space to copy the meta data. The space must be provided by the caller.
Returns
DYB_Ok - Success
DYB_OutOfRange - No full buffer available

Get Size of Data Frame.

The function returns the size of a complete data frame for the channel. This is the buffer size required for a call to DYB_getDataBuffer.

The size may vary when measurement parameters are changed. It is not valid before the data acquisition has started!

Write Buffer to file.

Writes a buffer (as retrieved with DYB_getDataBuffer) to a file of an appropriate ascii or binary format. The format is chosen automatically according to the meta data.

Available formats are "bcrf" (binary) and "asc" (ascii) for scanner triggered data and "csv" for all other data. The formats are "Daisy compatible".

Parameters
fileNameName of the file to write, without extension (selected automatically).
commentData or channel description for the file header. Can be left blank.
binaryIf the desired format is binary. Relevant only for scanner triggered data, ignored otherwise.
forwardIf the forward scan (in X direction) is to be written. Relevant only for scanner triggered data, ignored otherwise.
indexIndex of the first element in the buffer.
dataSizeNumber of valid data (32-bit items) in the buffer.
dataThe data buffer.
metaMeta data belonging to the buffer.
Returns
DYB_Ok - Success
DYB_OutOfRange - Desired format not available
DYB_OpenError - Can't write to file
