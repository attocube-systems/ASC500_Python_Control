Daisy

Daisy is a software package to control the ASC500 SPM Controller by a PC. It consists of the following parts:

  • A product server program that encapsulates most of the application logic. It communicates with the ASC500 over USB and with its client(s) via TCP/IP. It can handle several clients at the same time.
  • For booting the device the server requires several firmware files (*.bit, *.ldr).
  • Daisy(.exe), a generic user interface that is configured by loading "panels" (*.ngc files). A Panel consists of numerical or graphical control elements and data displays.
  • Profiles (*.ngp) that contain a set of control parameters and links to the corresponding panels. They can be seen as workspaces for daisy.
  • Daisybase (daisybase.dll / daisybase.so) is a library for custom programming of ASC500 applications. A daisybase application behaves as a client to the server just like Daisy. Therefore it is possible to run the application and Daisy in parallel.

Daisybase API

The custom programming interface of the daiysybase library is declared in the following C header files:

  • daisybase.h: Generic functions to handle the server, send commands to it and receive data from it.
  • daisydata.h: It contains a set convenience functions with emphasis on the handling of data streams. It allows a less flexible but simpler programming model than daisybase.h.
  • asc500.h: A headerfile that defines a set of constants to be used with the functions of daisybase.h (parameter identifiers, enum values).
  • metadata.h: Functions that help to interpret received data.

In order to run a daisybase application, make sure that the program loader can find the daisybase.dll/.so and the Qt libraries (Qt*.dll / .so).
